# Role Onboarding Checklist

Use this checklist whenever a team member is assigned to a project role. Complete each item before the person begins active delivery work.

Refer to [`docs/templates/role-responsibility-template.md`](../templates/role-responsibility-template.md) to document the role details that this checklist references.

---

## Access & Tooling
- [ ] Repository / codebase access granted (correct branch permissions)
- [ ] Project board access confirmed (GitHub Projects or equivalent)
- [ ] CI/CD pipeline credentials or service accounts set up
- [ ] Communication tools joined: Slack channels, mailing lists, calendar groups
- [ ] Required SaaS or internal tool licences provisioned (design tools, monitoring, etc.)

## Introductions & Context
- [ ] Introduced to Project Manager and Product Manager
- [ ] Introduced to Technical Lead and immediate team members
- [ ] Met with counterpart from any dependent team or external stakeholder
- [ ] Async introduction sent to the full project channel

## Documentation Review
- [ ] Project one-pager (problem statement, goals, success metrics) reviewed
- [ ] Role-responsibility document for this role reviewed (see template)
- [ ] Project planning doc and milestone map reviewed
- [ ] Risk register reviewed; risks relevant to this role noted
- [ ] Definition of Done and acceptance criteria standards reviewed
- [ ] Relevant architectural decision records (ADRs) or design docs reviewed

## Responsibilities & Handoffs
- [ ] Responsibilities confirmed with PM and direct collaborators
- [ ] Escalation path understood (Level 1 → 2 → 3)
- [ ] Interaction matrix reviewed: who this role hands off to and receives from
- [ ] PR / review conventions understood (branching strategy, review policy)

## Success Metrics
- [ ] Role-level goals and KPIs agreed with PM/PdM
- [ ] First milestone or deliverable identified
- [ ] Check-in cadence agreed (e.g., end of first sprint)

## Initial Tasks
- [ ] First backlog items assigned in project board
- [ ] Any blocking dependencies flagged to PM
- [ ] Calendar holds placed for recurring ceremonies (standup, planning, retro)
