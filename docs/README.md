# OctoAcme Project Management Docs

This folder contains the primary project management and program process documentation used by the OctoAcme team. These docs capture how we initiate, plan, execute, release, and continuously improve cross-functional projects. They are intended to accelerate onboarding, reduce single-person dependencies, and provide a single source of truth for project delivery practices.

Quick overview of our project lifecycle

1. Initiation — Define the problem, success metrics, stakeholders, and a high-level timeline so everyone starts aligned.
2. Planning — Break work into shippable increments, estimate effort, identify dependencies and risks, and create a release and milestone plan.
3. Execution & Tracking — Deliver work using a project board and PR workflow, track progress and risks, run tests and quality checks, and hold regular demos and syncs.
4. Release & Deployment — Follow release-type-specific checklists (patch/minor/major), run smoke tests, document release notes, and have rollback plans ready.
5. Retrospective & Continuous Improvement — Run retrospectives after sprints, releases, and incidents; convert learnings into action items and measure improvements.
